Регламент обновления аналитических дашбордов
Цель

Обеспечить единый процесс обновления дашбордов «План/Факт» и «Юнит-экономика», чтобы данные были актуальными и соответствовали источникам.

Когда обновляем

Обновление выполняется:

после получения новых данных;
по расписанию (например, раз в неделю);
по запросу бизнеса.

Источники данных

В демо-проекте данные формируются Python-скриптами и загружаются в PostgreSQL.

Основные этапы:

генерация/получение исходных данных (scripts);
загрузка данных в БД;
выполнение SQL-витрин;
экспорт итоговых данных в CSV;
обновление HTML-дашбордов.

Порядок обновления
1. Обновить исходные данные

Запустить Python-скрипты или загрузить новую выгрузку.

В результате должны обновиться CSV-файлы в папке data.

2. Обновить данные в PostgreSQL

Последовательно выполнить SQL-скрипты:

создание таблиц;
загрузка данных;
построение аналитических витрин.

Полученные витрины экспортировать обратно в CSV.

3. Пересобрать дашборды

Запустить Python-скрипты генерации HTML.

В результате должны обновиться:

plan_fact_dashboard.html;
unit_economics_dashboard.html.

4. Проверить результат

Перед публикацией необходимо убедиться, что:

KPI совпадают с расчетами в CSV;
корректно работают фильтры и поиск;
графики отображаются без ошибок;
HTML открывается локально без дополнительных настроек.

5. Публикация

После проверки обновленные HTML-файлы публикуются в общей папке или направляются заказчику удобным способом.

При необходимости предыдущая версия сохраняется в архив.

Если обнаружена ошибка

Если после публикации найдено расхождение:

использовать предыдущую рабочую версию;
исправить источник данных или SQL-запрос;
повторно пересобрать дашборды;
